# feishu-deck-h5 · 对外分享材料（厉害在哪里）

> 用途：对外讲清楚「这个技能到底和『给句话就出一套 HTML』有什么本质不同」。
> 两条主线：① 校验是怎么一层一层查的 ② 怎么把复杂度和能力「看得见」。
> 所有数字均已在仓库里核对（截至 2026-06-15）；带「据内部审计」的来自技能自带审计文档。

---

## 0. 一句话定位

> 别人是 **「一句话 → 一份 HTML」**：开环、一次性、改一处要重抽整页、不可维护。
> 这个技能是 **「结构化中间层（deck.json）+ 渲染器 + 一个会真的在浏览器里量像素的闭环校验引擎」**：闭环、可回改、有制度记忆。
>
> **任何人都能让 LLM 出一次 HTML；几乎没人有「量真实渲染像素的校验器 + 失败回滚的闸门 + 上百次事故沉淀的规则库」，让接下来的一千份 deck 可靠地正确。**

## 1. 核心反差：一次性出 HTML vs 闭环出片工厂

| | 一次性「出 HTML」 | feishu-deck-h5 |
|---|---|---|
| 产物 | 一坨 `index.html`，就是终点 | `deck.json` 是**唯一真相**，`index.html` 是**纯派生** |
| 改一处 | 重抽整页，祈祷别处不漂 | 改 JSON 字段 + 重渲，或浏览器编辑回灌 |
| 看历史 | 巨大 HTML diff，没法看 | JSON diff 可读、可评审 |
| 复用一页 | 复制粘贴 HTML，样式/资源全靠运气 | 按语义 `data-slide-key` 整对象 lift，连资源带 CSS 一起搬 |
| 正确性 | 「看着对」 | 重渲后**量真实渲染像素**，不过就回滚，进不了交付 |
| 一致性 | 逐页手感漂移 | 跨页一致性也有规则（标题位/配色/字阶）守 |

闭环：`deck.json` →（render-deck 派生 HTML）→（无头 Chromium 在 1920×1080 **量真实像素**）→（严重度归并）→（闸门：过就交付 / 不过就**回滚 index.html**，return 4）→ 回到 `deck.json`。
改一页只重量改动页（`--iter`，秒级）；定稿才跑全量 `--final`。

## 2. 校验是怎么「一层一层」查的（五层 gauntlet）

一份 deck 出门前，要穿过五层校验，从**最便宜的字节扫描**一路升到**最贵的真实像素测量**——由便宜到昂贵，早层先拦。

| 层 | 查什么 | 怎么查 | 成本 |
|---|---|---|---|
| **L0 · Schema 契约** | HTML 还没生成，先查源 `deck.json` 形状：JSON-Schema + Schema 表达不了的跨字段规则（key 唯一、表格行列对齐、agenda 单 active、canvas 几何…） | 纯 Python，**不开浏览器** | 毫秒 |
| **L1 · 字节 / 文件系统** | 浏览器会**偷偷补全**、因此渲染后 DOM 看不见的东西：文档截断、烤死的活 DOM、来源哈希漂移、性能预算（base64/blur/监听器…） | 正则扫**原始 HTML 字节** | 毫秒 |
| **L2a · 源码文本规则** | 无浏览器时的兜底：结构、违禁标点、slug 命名、转义标签泄漏 | 逐帧正则扫源码 | 毫秒 |
| **L2b · 真实像素引擎** ⭐ | 渲染态几何/计算样式/结构的全集：溢出、重叠、字号落 4 档阶梯、视觉焦点、留白均衡、死规则… | **无头 Chromium 渲染 → 切 present 模式 → 等动画落定 → 量 `getBoundingClientRect` ÷ 实际缩放**还原到设计像素 | ~1–5s |
| **L3 · 归并 + 严重度 + 报告** | 不加新规则：把发现倒进桶，`--strict` 升级、按页过滤，出工程师视图或「业务话术」逐页报告 | 进程内 Python | 可忽略 |

**关键点（对外一定要强调）：L2b 量的是「浏览器算出来的像素」，不是源码字符串。** 这正是 `validate.py` 反复把「看着没问题、其实全黑 / 16px / 背景丢了」的 deck 报 PASS 的根因——只有量真实像素才能补上这条缝。这是别人没有的一层。

### 一套引擎，两条信号路，强制对齐
- 只有**一套规则引擎**：`audits.js`（DOM/像素，跑在 Chromium）+ `run-audits.py`（字节/文件系统），由 `validate.py` 用 `run_unified_audits` 合并；`--no-visual` 只是去掉 Chromium 那半，没有第二套规则。
- 规则**三处同源**：引擎 ↔ `business-rules.yaml`（业务话术）↔ `check-only.py`（工程师视图），还有一个 `check-rule-coverage.py` **防漂移硬闸**保证三处永不脱节。
- 静态扫描和渲染像素**共享同一套词表**（chrome 类名、4 档字阶 `{16,24,28,48}` 都从 `--fs-*` token 运行时派生），并有 parity 测试锁死两边相等。

### 严重度模型
`err`（硬拦）/ `warn`（默认放行，`--strict` 升 error）/ `warn_soft`（编辑建议，**永不升级**——所以「方向性提醒」不会训练大家忽略闸门）。还有**按来源分级**：同一个缺陷，lift/导入来的判 error，作者自己写的判 warn——忠实复刻别人的 deck 不会被一堆「不是你引入的」红字淹没。

### 三道闸门（什么时候是硬拦）
1. **渲染前**：`render-deck.py` 先 `validate-deck.py --strict` 查 `deck.json` Schema，坏的根本不渲。
2. **渲染后**：对刚渲的 HTML 跑 `validate.py`，错了就**回滚**，残缺 deck 进不了交付。
3. **入库前**：`check-only.py --gate ingest` 强制 strict + visual，过滤到约 21 条「业务必修」码，外加包契约（资源齐全、无符号链接/越级路径）。

## 3. 校验把规则映射到「客户视角」的三个桶

`business-rules.yaml` 把每条技术规则翻译成业务后果——这是对外讲「为什么值钱」的黄金叙事：

- **A · 客户看不见**：投影上字太小 / 字色融背景 / 内容堆到顶被裁。
- **B · 库找不回这张片**：缺身份标识 / DOM 坏了 / key 重复——切片不可索引。
- **C · 复用时会打架**：字号失控 / 品牌色错 / 排版逐页漂移。

## 4. 战场规则精选：一次性 LLM 永远抓不到的 bug

每条规则背后都是一次**真实事故**（约 327 个修复工单）。挑最有说服力的几条对外讲：

- **R-DOC-INTEGRITY** — 一次 sed 吃掉了 `.deck` 收尾 `</div>`，文档截断。浏览器自动补全标签，DOM 看着「干净」、旧校验报 PASS——可投影上整页全黑。*一次性出 HTML 抓不到：解析器会静默补全不平衡标签，必须数原始字节才看得见。*
- **R-BAKED-DOM** — 「烤死的活 DOM」（浏览器另存 / 编辑态保存）再发布会二次初始化 JS，页码定格 `01/01` 但还能翻页，翻到第 5 页才有人发现。*抓不到：烤出来的标记语法完全合法、看着完整，缺陷只相对渲染管线不变量存在。*
- **R-VIS-DEAD-ANIM** — 正则把合法的 `.slide-frame.is-current` 啃成 `-frame.is-current`（合法 CSS 标识符，能解析），但没有 `<-frame>` 元素 → 选择器零匹配 → 入场动画静默死、内容永久隐身。*抓不到：静态读 CSS 完全合法，bug 纯属运行时，必须在正确作用域查活 DOM。*
- **R-VIS-DEAD-RULE** — `.hero-pct:100px` 死选择器退回继承的 16px，而 16 是合规阶梯档，所有字号闸全绿、零报警。*抓不到：失败落在一个合规默认值上，要把「声明了但够不着」对照活 DOM 才知道意图样式从未生效。*
- **R-VIS-CARD-OVERFLOW** — 密集卡片静默裁字；更隐蔽的是 `justify-center` 卡**顶沿**溢出——`scrollHeight` 结构上报不了框上方的内容。*抓不到：溢出取决于特定缩放下的渲染高度，从标记算不出来。*
- **R-CSS-CROSS-PAGE** — 这页的 `<style>` 写了**别页**的 key（华泰案例：一页 `<style>` 涨到 ~1.5MB）；删/搬这页会静默搞坏另一页，没有报错。*抓不到：跨文档、延时的耦合，评审单页 HTML 没有这个心智模型。*
- **R-AUTOBALANCE-PRESENT** — 缺当前 `feishu-deck.js` → 自动配平运行时**跑 0 行**（青啤 raw deck 实测）→「文字贴底」永不自愈。*抓不到：缺的是一段「加载后该 mutate DOM」的行为运行时，不是静态标记里的破绽。*
- **R-PROVENANCE** — `index.html` 与同目录 `deck.json` 哈希对不上 = 真漂移（手改/没重渲）。*抓不到：LLM 没有「这份 HTML 必须能从那份 JSON 字节推出」的跨进程概念；手改得「看着完美」本身就是缺陷。*
- **R-FOREIGN-SCRIPT** — lift 外来页拖进的 `<script>` 会随入库扩散、在飞书登录态 viewer 里变成内部观众浏览器的 XSS。*抓不到：一次性出 HTML 把外来素材当「要忠实复刻的内容」，没有「数据 vs 代码」「按来源分级」的安全模型。*
- **R-IFRAME-REMOTE** — 远程 iframe 离线/无登录现场静默变白块，还让截图工具每帧耗尽 30s 超时。*抓不到：开发期有网有登录一切正常，缺陷是运行时/环境依赖。*

## 5. 护城河 / 硬数据（已核对）

- **70+ 条校验规则**（统一引擎 72 个规则 ID），其中 **32 条**在无头浏览器里量真实渲染像素；文档登记 **93 条**规则条目，**60+** 条镜像成业务话术。
- **3 条跨页一致性规则**（标题位漂移 / 配色漂移 / 字阶预算）——单页规则看不见的「逐页漂移」。
- **~327 个修复工单**（`F-001…F-332`），**369 次提交 / 6 周**（5/1–6/14）。
- **96 个测试文件 / 789 个测试用例**，含规则 parity 防漂移闸。
- 工程量：Python **~2.4 万行** + 审计引擎 JS **~7 千行**；**9 个子技能、12 种路由模式、25 篇参考文档**。

据内部审计文档（来自技能自带的 AUDIT / BENCHMARK 文件，对外可引用）：
- 正确性靠**多 agent 代码评审**而非感觉：2026-06-14 一轮跑 **68 个 agent / 13 个子系统**，**87 条原始发现里 86 条经独立复现确认**（1 条被对抗性证伪）。
- 质量是**盲评基准**出来的：5 个 brief × {Opus, Sonnet} = 10 份真实 deck，**20 个独立盲评 agent** 按 5 个维度打分——结论是**系统（schema/模板/校验器）而非模型**扛起大部分质量，把前沿模型差距收到约 5%。
- PPTX 导入策略基于实测：**11 份真实 deck / 224 页**里，结构上无法重建的比例实测为 **0%**。
- 往返编辑（渲染 → 浏览器改 → 回灌 → 重渲）实测在文本/几何/增删/重排上**字节无损**。

## 6. 可直接转发的三段话术

**朋友圈 / 短**
> 让 LLM 出一份 PPT 网页谁都会。我做的是一条出片流水线：deck.json 当唯一真相，HTML 只是派生；出片后**真的开一个无头浏览器在 1920×1080 量渲染像素**，70+ 条规则（32 条像素级）逐层拦，不过就回滚，进不了交付。每条规则背后都是一次真实事故，攒了 ~327 个修复工单、近 800 个测试用例。

**工作群 / 中**
> 核心差别一句话：别人开环出一坨 HTML，我是闭环工厂。五层校验从字节扫到真实渲染像素——能抓到一类「源码完全合法、但浏览器里其实全黑 / 字号退默认 / 两块文字叠一起 / 删一页搞坏另一页」的 bug，这些一次性出 HTML 的做法**结构上**抓不到，因为它们只在布局后、跨进程、或运行时才显现。规则还映射成客户视角三桶（看不见 / 库找不回 / 复用打架），和近 800 个测试一起把每次踩的坑都焊死成回归。

**正式介绍 / 长** → 见本文第 0–5 节。

## 7. 现场演示提示词（让别人「看见」厉害）

把这些当场跑给观众看，比讲 PPT 有说服力：

1. **「制造一个 bug，看校验抓不抓得到」**
   > 把这份 deck 的某页改成 18px 正文 / 把 `.deck` 收尾 `</div>` 删掉 / 把一段动画选择器改成 `-frame.is-current`，然后跑校验，看它报什么、怎么教我修。
2. **「让校验解释自己」**
   > 用「逐页业务报告」模式校验这份 deck，按「客户看不见 / 库找不回 / 复用打架」三桶给我列问题和修法。
3. **「闭环改一页」**
   > 只改第 7 页的标题和一张卡，`--iter` 重量这一页，告诉我用了几秒、动了哪些文件、校验过没过。
4. **「lift 一页过来」**
   > 从另一份 deck 把某张关键页 lift 进来，跑资源引用扫描 + lift 体检，看它怎么把 CSS 预算和死规则收拾干净。
5. **「元演示」**
   > 用这个技能给「这个技能本身」做一份对外分享的 16:9 演示 deck（自举/吃自己的狗粮）。
