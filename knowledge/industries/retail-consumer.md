# 行业包 · 消费零售 / 连锁餐饮

适用:连锁餐饮、商超便利、美妆服饰、新消费品牌、区域零售集团。

## 高价值业务时刻

| 业务时刻 | 常见目标 | 关键受众 |
|---|---|---|
| 新品上市 | 缩短从总部策略到门店执行的链路 | 增长负责人、运营负责人 |
| 门店巡检 | 让标准动作可追踪、可复盘、可纠偏 | 区域经理、门店运营 |
| 会员复购 | 打通活动、私域、门店和客服信息 | CRM、市场负责人 |
| 人员训练 | 降低新人上手成本,复制优秀门店经验 | 培训负责人、HRBP |
| 经营看板 | 把销量、库存、活动、人员动作放到同一视图 | CEO、COO、财务 |

## 痛点地图

| 痛点 | 业务后果 | 证据建议 | 适合页型 |
|---|---|---|---|
| 总部策略到门店动作衰减 | 活动执行不一致,复盘只看到结果看不到动作 | 活动 SOP、巡店记录、门店反馈截图 | `flow/process`, `content/before-after` |
| 数据散在系统和群聊里 | 决策滞后,区域问题不能快速定位 | BI 截图、群聊样例、报表字段 | `arch-stack`, `content/matrix` |
| 一线问题沉淀不回知识库 | 优秀经验无法复制,新人培训靠师傅口述 | FAQ、培训材料、质检记录 | `content/3up`, `flow/tree` |
| 客户反馈和经营动作脱节 | 市场活动难以快速改口径、改投放 | 客服工单、社媒评论、门店反馈 | `content/story-case`, `quote` |
| 门店管理依赖人工盯群 | 管理成本高,异常发现慢 | 群数量、巡检频率、人工汇总耗时 | `stats/row`, `stats/hero` |

## 推荐主张

1. **从“群聊驱动执行”转成“流程 + 数据 + 知识库驱动执行”**。
2. **让 AI agent 承担巡检、答疑、复盘、提醒等重复管理动作**。
3. **把门店现场、会员反馈和总部策略放在同一张经营地图里**。

## 素材提示

- 客户 logo:优先查 `assets/shared/clientlogo/`。
- 门店 / 商品 / 现场照片:优先用户提供,不要用泛 stock photo 伪装真实现场。
- 产品 icon:优先用 `assets/shared/feishu-products/`。
- demo:如果用户要求展示 bot / agent 使用路径,用 `iframe-embed` 或 `phone-iframe`
  嵌入真实 demo,不要只贴静态截图。

## 常见 open questions

- 这次 deck 是面向客户高层、业务负责人,还是内部方案评审?
- 有无真实门店数据、活动 SOP、群聊样例、BI 截图可引用?
- 是否要强调本地 agent 使用,还是飞书 bot 内的闭环体验?
- 输出是客户提案、内部汇报、产品方案,还是案例复盘?
